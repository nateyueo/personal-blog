## PRD 的核心要素

一份合格的 PRD 至少需要包含以下要素：

### 1. 背景与目标

明确回答「为什么要做这个产品/功能」：

```markdown
## 项目背景
[描述市场环境、用户痛点、业务目标]

## 项目目标
- 目标 1：可量化的业务指标
- 目标 2：可验收的功能交付
```

### 2. 用户画像

描述核心用户群体及其需求：

- **谁**：用户角色与特征
- **要什么**：核心需求与期望
- **在什么场景下**：使用场景与上下文

### 3. 功能需求

按优先级分层的功能清单：

| 优先级 | 功能 | 验收标准 |
|--------|------|----------|
| P0 必须 | 核心功能 | 明确的验收条件 |
| P1 重要 | 增强功能 | 有则更好 |
| P2 可选 | 锦上添花 | 后续迭代 |

### 4. 非功能需求

性能、安全、可用性等约束条件。
