## 什么是 PRD 手册？

**PRD（Product Requirements Document）** 即产品需求文档，是产品经理最核心的输出物之一。

本手册旨在系统化地沉淀 PRD 编写方法论，包括：

- **流程**：从需求收集到文档交付的完整流程
- **模板**：可复用的 PRD 模板与组件
- **清单**：自查清单与评审要点
- **示例**：真实案例与最佳实践

## 适用人群

| 角色 | 价值 |
|------|------|
| 产品经理 | 系统学习 PRD 方法论，提升文档质量 |
| 设计师 | 理解需求文档结构，优化交付协作 |
| 开发工程师 | 更好地理解产品意图，减少返工 |
| 项目经理 | 掌握需求管理最佳实践 |

1. **新手** — 从「快速开始」章节入门
2. **有经验者** — 直接跳到感兴趣的专题章节
